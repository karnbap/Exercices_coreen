이전 코밋으로 되돌리기

Comment revenir à un commit précédent

Git 버전 관리 되돌리기 완전 가이드 / Guide complet pour revenir en arrière avec Git

git reset git revert git checkout 시나리오 / Scénarios

소개 / Introduction

한국어 🇰🇷

Git에서 이전 상태로 되돌리는 방법에는 여러 가지가 있습니다. 각각의 방법은 다른 상황에서 사용되며, 작업 내용을 잃지 않고 안전하게 되돌리는 것이 중요합니다.

Français 🇫🇷

Il existe plusieurs façons de revenir en arrière avec Git. Chaque méthode est utilisée dans différentes situations, et il est important de revenir en arrière en toute sécurité sans perdre votre travail.

git reset 사용법

⚠️ 주의사항 / Attention

한국어: git reset은 커밋 히스토리를 변경합니다. 이미 푸시한 커밋에는 사용하지 마세요!
Français: git reset modifie l'historique des commits. Ne l'utilisez pas sur des commits déjà poussés !

1. Soft Reset (작업 내용 보존 / Préserver le travail)

git reset --soft HEAD~1
# 마지막 커밋을 취소하지만 변경사항은 staging area에 보존
# Annule le dernier commit mais préserve les changements dans le staging

2. Mixed Reset (기본값 / Par défaut)

git reset HEAD~1
# 커밋과 staging을 취소하지만 작업 디렉토리는 보존
# Annule le commit et le staging mais préserve le répertoire de travail

3. Hard Reset (모든 것 삭제 / Tout supprimer)

git reset --hard HEAD~1
# 모든 변경사항을 완전히 삭제 (위험!)
# Supprime complètement tous les changements (dangereux !)

git revert 사용법

✅ 안전한 방법 / Méthode sûre

한국어: git revert는 새로운 커밋을 만들어서 이전 변경사항을 되돌립니다. 히스토리를 보존하므로 협업시 안전합니다.
Français: git revert crée un nouveau commit qui annule les changements précédents. Il préserve l'historique, donc sûr pour la collaboration.

특정 커밋 되돌리기 / Annuler un commit spécifique

git revert <commit-hash>
# 예시: git revert abc123
# Exemple: git revert abc123

마지막 커밋 되돌리기 / Annuler le dernier commit

git revert HEAD
# 가장 최근 커밋을 되돌리는 새 커밋 생성
# Crée un nouveau commit qui annule le commit le plus récent

여러 커밋 되돌리기 / Annuler plusieurs commits

git revert HEAD~3..HEAD
# 최근 3개 커밋을 각각 되돌리는 커밋들 생성
# Crée des commits qui annulent les 3 derniers commits

git checkout 사용법

ℹ️ 임시 확인 / Vérification temporaire

한국어: git checkout은 이전 상태를 임시로 확인할 때 사용합니다. 영구적인 변경은 아닙니다.
Français: git checkout est utilisé pour vérifier temporairement un état précédent. Ce n'est pas un changement permanent.

특정 커밋으로 이동 / Aller à un commit spécifique

git checkout <commit-hash>
# 특정 커밋의 상태로 임시 이동 (detached HEAD)
# Se déplace temporairement vers un commit spécifique (detached HEAD)

특정 파일만 되돌리기 / Restaurer un fichier spécifique

git checkout HEAD~1 -- filename.txt
# 특정 파일만 이전 버전으로 복원
# Restaure seulement un fichier spécifique à une version précédente

현재 브랜치로 돌아가기 / Retourner à la branche actuelle

git checkout main
# 또는 git switch main (최신 Git 버전)
# ou git switch main (versions Git récentes)

상황별 가이드 / Guide par scénario

🔄 시나리오 1: 방금 커밋한 내용을 수정하고 싶을 때

한국어: 커밋 메시지를 잘못 작성했거나 파일을 빼먹었을 때
Français: Quand vous avez mal écrit le message de commit ou oublié un fichier
git reset --soft HEAD~1
# 변경사항 수정 후
git add .
git commit -m "올바른 커밋 메시지"

🚫 시나리오 2: 이미 푸시한 커밋을 되돌리고 싶을 때

한국어: 다른 사람들과 공유한 커밋을 안전하게 되돌리기
Français: Annuler en toute sécurité un commit partagé avec d'autres
git revert <commit-hash>
git push origin main
# 새로운 "되돌리기" 커밋이 생성됨

🔍 시나리오 3: 이전 버전을 확인만 하고 싶을 때

한국어: 이전 상태를 보고 싶지만 변경하고 싶지는 않을 때
Français: Quand vous voulez voir l'état précédent sans le modifier
git log --oneline
# 커밋 해시 확인 후
git checkout <commit-hash>
# 확인 후 돌아가기
git checkout main

  • ✅ git revert - 협업시 사용 / Pour la collaboration

  • ⚠️ git reset --mixed - 로컬에서만 / Local seulement
